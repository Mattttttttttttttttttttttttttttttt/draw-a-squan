Important interpretation:
The CPU is not merely idle or RAM-bottlenecked. It is under full load but remains stuck at about 1.09 GHz. Linux detects turbo support and the 2.8 GHz max, but actual runtime frequency never boosts. This strongly suggests a Linux CPU frequency/pstate/firmware issue, not normal performance.

Most suspicious finding:
intel_pstate/status = passive
driver = intel_cpufreq

The previous assistant suspected passive intel_pstate / intel_cpufreq behavior may be failing to trigger turbo properly on this low-end Intel laptop. The next planned fix was to try forcing active intel_pstate in GRUB.

Suggested next step:
Edit /etc/default/grub and add:
intel_pstate=active

Example:
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash intel_pstate=active"

Then run:
sudo update-grub
sudo reboot

After reboot, check:
cat /sys/devices/system/cpu/intel_pstate/status

Desired output:
active

Then stress test:
stress -c 2

Monitor:
watch -n0.5 "grep 'cpu MHz' /proc/cpuinfo"

Expected if fixed:
CPU should boost above 1.09 GHz, maybe 1.8–2.8 GHz depending on thermal/power limits.

Fallback if intel_pstate=active does not fix it:
Try:
intel_pstate=disable

This should force acpi_cpufreq instead of intel_cpufreq. Some cheap Intel laptops behave better with acpi_cpufreq.
